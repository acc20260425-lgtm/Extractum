# Постоянный инспектор + рабочий футер (/projects/next) — design

**Дата:** 2026-07-03
**Статус:** согласовано (brainstorming), готово к плану реализации
**Область:** страница `/projects/next` (проводка); компонент `Inspector.svelte`
НЕ меняется — он уже реализует v10 полностью (открытое состояние 300px,
свёрнутый рейл 44px с вертикальной подписью «Инспектор», футер
Синхронизировать/Отключить, `selected: InspectorSource | null`).
**Эталон:** `Inspector.dc.html` + `Research Projects v10.dc.html`
(колонка инспектора постоянна, `inspWidth` 301px/45px).

## Проблема

`inspector`-бэг передаётся в shell только при активном источнике
(`inspectorSource ? {...} : undefined`) — без него колонка исчезает целиком.
Кнопки футера отрисованы, но `onSync`/`onDisconnect`/`syncDisabled` со
страницы не проведены — мёртвые.

## Решение (только `src/routes/projects/next/+page.svelte`)

### 1. Постоянная колонка

`inspector`-бэг передаётся всегда при выбранном проекте:

- `selected: inspectorSource` (может быть `null` — компонент показывает
  «Конфигурацию проекта», футер скрыт);
- `open: inspectorOpen`, `onToggle` — уже есть (сворачивание в 44px работает);
- Без выбранного проекта колонки нет (экран «Выберите проект») — как сейчас.

### 2. Футер

- `activeSyncable` (derived): активный источник существует и
  `provider === "youtube" && source_subtype ∈ {"video","playlist"}`
  (тот же предикат, что у bulk-sync).
- `syncDisabled: railState.saving || !activeSyncable`.
- `onSync`: `syncYoutubeSource(Number(activeSourceId), { metadata: true,
  transcripts: true, comments: false })` → `sources = await
  listProjectSources(...)`; ошибки → `railState.status`
  (`Не удалось синхронизировать источник (…)`); `saving` на время операции.
- `onDisconnect`: открывает `ExtractumDialog` подтверждения на странице
  («Отключить источник «{title}» от проекта? Материалы останутся в
  библиотеке.», кнопки «Отмена» / «Да, отключить» (destructive)).
  По подтверждению: `removeProjectSources({ projectId, sourceIds:
  [Number(activeSourceId)] })` → `activeSourceId = null`, удалить id из
  `selectedSourceIds`, перезагрузить источники + `workflow.reload()`;
  ошибки → `railState.status` (`Не удалось отключить источник (…)`).

Состояние диалога: `disconnectOpen: boolean` (страница); имя источника —
из `selectedSourceRow?.title`.

## Тестирование

- Компонент уже покрыт (`Inspector.test.ts`); чистые функции не добавляются.
- Живая проверка в Tauri:
  1. проект выбран, источник НЕ активен → колонка видна («Конфигурация
     проекта», футера нет);
  2. «Свернуть» → 44px-рейл с вертикальной подписью; «Развернуть» — обратно;
     состояние переживает переключение источников;
  3. активный youtube-video источник → «Синхронизировать» активна; клик —
     запускается sync (статусы/логи), грид перезагружается;
  4. «Отключить» → диалог; «Отмена» не удаляет; «Да, отключить» — источник
     исчез из таблицы, инспектор пуст (селект сброшен), счётчики рейла
     обновились; восстановить источник через «Добавить источник»;
  5. чекбокс-выделение и сортировка не затронуты.

## Границы/изоляция

Меняется только страница; `Inspector`, shell, обёртки — без изменений.
`ExtractumDialog` — уже используемый паттерн подтверждения.

## Не в скоупе

Синхронизация не-youtube источников; изменение самого Inspector-компонента;
persistence `inspectorOpen` между сессиями.
